# Prompt Sequence Demos

按顺序使用这些提示词，把一个主题稳定做成参考图那种“中文手绘白板流程卡”。

## Demo 1: 选题提炼

```text
你是中文自媒体图解策划。请把主题「用好 Coding Agent」提炼成一张手绘白板风图解板的核心表达。

要求：
1. 给出 5 个标题方向，每个标题 10-22 个字。
2. 每个标题必须有一个明确判断，不要空泛。
3. 标题里允许出现中英混排，如 Coding Agent / Plan / Review。
4. 给出你最推荐的 1 个标题，并说明适合画成图的原因。

参考风格：手写笔记、编号步骤、彩色分区、箭头流程、底部一句话总结。
```

## Demo 2: 内容拆解

```text
围绕标题「用好 Coding Agent，重点是两头」，拆成一张竖版手绘白板图解的内容结构。

请输出：
1. 顶部标题和副标题。
2. 6 个编号模块，每个模块包含：
   - 模块小标题，4-10 个字
   - 2-4 条短句，每条不超过 18 个字
   - 适合配的小图标
3. 底部“这像什么？”类比总结，用 3 个角色串起来。

写法要求：
- 先讲结论，再讲步骤。
- 文案像老师在白板上写重点，不像正式 PPT。
- 每个模块只讲一个判断。
```

## Demo 3: 版式草图

```text
把下面内容设计成一张 2:3 竖版手绘白板风图解板的版式方案。

内容：
{粘贴 Demo 2 生成的标题、模块、总结}

请输出：
1. 整体版式说明，从上到下描述。
2. 每个模块的颜色、边框、图标、箭头关系。
3. 哪些词应该用红色强调或下划线。
4. 哪些地方适合加便签、星星、虚线回路、放大镜等手绘元素。

风格目标：温暖米白纸张、黑色马克笔线条、淡彩分区、可爱小机器人、课堂白板笔记感。
```

## Demo 4: 最终出图 Prompt

```text
生成一张中文手绘白板风技术图解长图，主题是「用好 Coding Agent，重点是两头」。

画面比例：2:3 竖版。温暖米白纸张背景，轻微纸纹，整体像手绘课堂白板笔记。
顶部大标题：「用好 Coding Agent，重点是两头」，黑色手写马克笔风；关键词「两头」用红橙色强调并加下划线。
副标题：「先把方向定对，后面才跑得稳」。

主体从上到下排列 6 个编号圆角分区，左侧有彩色编号圆点，区块之间用橙色向下箭头连接：
1. 「核心原则」：开头先定方向，结尾认真 Review；配小机器人和小旗子。
2. 「先做 Plan」：先让多个 Agent 各写方案，再比较；配三张方案卡片。
3. 「比较方案」：看可行性、风险、成本、缺口；配放大镜和星标。
4. 「按复杂度执行」：简单任务直接做，复杂任务分 Phase；配流程文件和勾选框。
5. 「推进纠偏」：每个 Phase 后人工看一眼，越早纠偏成本越低；配人物检查和箭头流程。
6. 「最后 Review」：不用太多 Agent，重点看需求、代码和测试；配代码窗口和放大镜。

底部总结条：「像组建小团队：顾问出方案，你拍板，执行者落地，架构师 Review。」画 3 个可爱人物和 1 个机器人，用小箭头串起来。

视觉风格：黑色手绘线稿、淡黄/浅蓝/浅绿/浅橙/浅紫分区、红色重点字、手绘星星、便签、虚线箭头、圆角边框、轻微不规则线条。
要求：中文尽量清晰可读，信息密集但留白舒服，结构清楚，禁止深色科技海报、霓虹、3D、写实摄影、企业 PPT 风。
```

## Demo 5: 修图迭代

```text
请基于上一张图继续优化，不改变主题和整体风格。

修改要求：
1. 保留手绘白板、米白纸张、编号流程、彩色分区。
2. 让标题更醒目，红色关键词只保留 1-2 个。
3. 模块之间留白更均匀，箭头更清楚。
4. 小图标减少一点，不要盖住正文。
5. 中文小字如果难以保证准确，就改成更短的关键词和示意线条。
6. 底部总结条保留，但压缩成一句话。
```

## Demo 6: 少字版 Prompt

中文小字不稳定时，先用这个少字版出图，再后期叠准确文案。

```text
生成一张手绘白板风中文技术图解海报模板，主题是「用好 Coding Agent」。

画面比例 2:3 竖版，米白纸张背景，黑色马克笔手绘线条。顶部有大标题区，主体有 6 个从上到下排列的彩色编号圆角模块，左侧编号 1-6，模块之间有橙色箭头。每个模块里只放 1 个短标题和几条模拟短线，不要生成大段小字。

元素包括：小机器人、方案卡片、放大镜、流程文件、人物检查、代码窗口、勾选框、星星、便签、红色下划线。

风格：可爱、清晰、信息图、课堂白板笔记、淡彩水彩填充、轻微纸纹。不要深色背景、不要霓虹、不要 3D、不要企业 PPT。
```
