# Handdrawn Explainer Board

把复杂主题整理成中文手绘白板风图解板：标题醒目、编号分区、彩色圆角框、箭头串联、手写感标注、小图标辅助说明，适合 AI/技术/方法论类自媒体图文。

## Workflow

1. 先判断用户要哪种输出：
   - `文案`: 返回标题、副标题、分区标题、每区短句、底部总结。
   - `版式`: 返回画面结构、分区顺序、元素摆放。
   - `提示词`: 返回可直接给图像模型的 prompt。
   - `出图`: 先生成文案和出图 prompt，再调用图像生成工具。
   - `技能模板`: 输出或更新 `SKILL.md` / prompt demo / reference docs。
2. 提炼主题的一句话结论：
   - 用 `主题不是 X，而是 Y`、`关键在 A 和 B`、`先做 A，再做 B` 这类可视化强的句式。
   - 标题控制在 `10-22` 个字；重点词可用红色强调。
3. 拆成 `5-7` 个编号模块：
   - 第 1 块放核心原则或总判断。
   - 中间块按流程、对比、选择、执行、复盘展开。
   - 最后块放总结、类比、行动建议或常见误区。
4. 给每块配置视觉角色：
   - 编号圆点：左侧纵向排列，用 `1/2/3...`。
   - 内容框：浅色圆角卡片，边框颜色各不相同。
   - 小图标：机器人、文件、放大镜、天平、流程箭头、人物、勾选框。
   - 强调元素：红色下划线、星星贴纸、便签、气泡注释、虚线回路。
5. 生成最终 prompt 时包含四层信息：
   - `内容层`: 标题、分区标题、每区要表达的短句。
   - `版式层`: 竖版长图、顶部标题、编号流程、底部总结。
   - `风格层`: 米白纸张、黑色马克笔、轻微手绘抖动、水彩淡色块。
   - `质量层`: 中文可读、信息密集但留白、不要暗色科技海报、不要企业 PPT 风。

## Visual DNA

- 画布优先用 `2:3` 或 `4:5` 竖版；复杂步骤可用长图。
- 背景是温暖米白纸张，带轻微纸纹，不用纯白或深色背景。
- 主标题像手写黑色马克笔，关键词用红橙色标出。
- 分区像课堂白板笔记：彩色边框、淡色底、左侧编号、区间箭头。
- 插图要可爱但克制，只服务解释，不抢正文。
- 底部可放一个“这像什么？”类比总结条，帮助读者记住。
- 参考风格锚点见 `assets/reference-coding-agent-board.png`，只学阅读节奏和视觉语言，不逐元素照搬。

## Copy Rules

- 每个模块只讲一个判断，不把长段文字塞进图里。
- 小标题像笔记分区，不像 PPT 章节，例如 `先做 Plan`、`比较多个方案`、`按复杂度执行`。
- 句子优先短、直、可复述：`越早纠偏，成本越低`。
- 比较类内容写成 `方案 A / 方案 B / 方案 C -> 比较 -> 优化`。
- 技术词可以保留，但必须配一句人话解释。
- 如果要让模型直接生成中文文字，提醒中文小字可能不稳定；高要求场景优先生成空版式或少字版，再后期叠加准确文字。

## Prompt Assembly

最终出图 prompt 使用这个骨架：

```text
生成一张中文手绘白板风技术图解长图，主题是「{主题}」。

画面比例：{2:3 或 4:5} 竖版，温暖米白纸张背景，轻微纸纹。
顶部大标题：「{标题}」，手写黑色马克笔风，关键词「{关键词}」用红橙色强调并加下划线。
副标题：「{副标题}」。

主体是 {5-7} 个从上到下排列的编号圆角分区，每区左侧有彩色编号圆点，区块之间用橙色向下箭头连接。
每个区块有不同淡色边框和浅色底：
1. 「{模块1标题}」：{模块1短句}，配 {图标1}
2. 「{模块2标题}」：{模块2短句}，配 {图标2}
3. 「{模块3标题}」：{模块3短句}，配 {图标3}
4. 「{模块4标题}」：{模块4短句}，配 {图标4}
5. 「{模块5标题}」：{模块5短句}，配 {图标5}

底部总结条：「{一句话总结}」，旁边画 2-3 个可爱人物/机器人小插图。

视觉风格：手绘线稿、黑色马克笔、淡彩水彩填充、便签、星星、虚线箭头、放大镜、勾选框、小旗子。
要求：中文尽量清晰可读，布局整洁，信息密集但不拥挤，禁止深色科技海报、霓虹光效、3D 渲染、企业 PPT 风。
```
